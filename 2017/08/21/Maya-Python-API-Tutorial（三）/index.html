前言

上個章節提到了如何傳flag參數，其實Flag中還可以包含多個參數，包成一個tuple的資料結構傳遞。

在一個Flag中包許多參數有什麼好處呢？
通常能解決程式邏輯的完整性還有可讀性，比方說RGB Color通常會把R、G、B包在一起傳遞，成為一個類似cmds.myCommandWithTupleFlag( Color = (255, 255, 255) )的Command，如果我們分別把R、G、B分開來，邏輯上就不能表示Color了，也看不出來R、G、B到底是做什麼的。
