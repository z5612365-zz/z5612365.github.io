Pycharm IDE+Auto Complete取代Maya原生Editor

發表於 2017-08-27 |

前言

原本是想要簡單的在其他Editor中來編輯Maya python command，因為Maya內建的Editor實在是讓我看的很不順眼。

不過從在其他Editor中編輯Maya python Code的想法的令一個延伸，是不是有辦法不開Maya就執行Maya python Code？
可以參考這篇文章

Auto Complete

https://forums.autodesk.com/t5/maya-programming/maya-python-api-2-0-ide-code-complete/td-p/6239681

  1. 先去這裡https://apps.exchange.autodesk.com/MAYA/en/Home/Index
    抓Maya2017_Update3_DEVKIT_Linux.tgz
    解壓縮後會有devkitBase資料夾
    把devkit複製到/usr/autodesk/maya2017裏面

  2. 然後到Pycharm設定mayapy作為直譯器’/usr/autodesk/maya2017/bin/mayapy’
    接下來增加其中的path ： /usr/autodesk/maya2017/devkit/other/pymel/extras/completion/py

  3. 刪除path中的/usr/autodesk/maya2017/lib/python2.7/site-packages ，因為會影響auto complete

  4. 大功告成！

自幹控

也可以不用MayaCharm，如果你是個自幹控可以這樣做！
http://forums.cgsociety.org/archive/index.php?t-966699.html

要關閉的話就變成 cmds.commandPort(name=":54321", sourceType="python", close=True)

其中name=”:54321”代表我們在Maya所開的port，sourceType=”python”代表傳入的是python command，若是sourceType=”mel”就代表傳入的是mel command。
