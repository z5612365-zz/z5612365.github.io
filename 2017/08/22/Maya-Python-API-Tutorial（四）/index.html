發表於 2017-08-22 |

前言

這個章節討論如何印出DAG Nodes Paths。
The Directed Acyclic Graph (DAG)是Dependency Graph的一部份
Dependency Graph其中包含了很多種node types(shaders, deformers, constraints…)，不過這個章節只Focus在DAG nodes。

  1. Transform Nodes (MFn.kTransform)
    定義了一個local 4x4 transformation matrix影響階層結構其下的所有nodes。
    Transform Nodes可以作為Transform Nodes可以作為的下層。

  2. Shape Nodes (MFn.kMesh, MFn.kCamera, MFn.kLight, … )
    包含了actual geometric information of a scene element。
    Shape Nodes一定要有Transform Nodes作為其上層。

  • world > scene’s root
  • green circles(kTransform) > transform nodes : 能夠讓shape nodes被放到scene
  • blue circles > shape nodes
    • perspShape(kCamera) > camera
    • pCubeShape1(kMesh) > cubic mesh
    • pointLightShape1(kPointLight) > point light

因為pointLight1是在pCube1的下層，所以如果pCube1移動的話，pointLight1也會跟著移動。

一般來說shape node下層就不該有東西了，不過有個特殊例外叫作underworld，underworld是DAG的subgraph，並用來control points and dependencies of a NURBS curve or a NURBS surface.

DAG Paths

  • MDagPath.node() > return the MObject to which it corresponds in the DAG
  • MDagPath.extendToShape() > be extended to encompass a shape node
  • MDagPath.transform() > return the lowest transform node
  • MDagPath.fullPathName() > returns a string representation of the DAG path to a given node
    每個node由pipe-separated (“|”)分開，並從root開始，不過root node並沒有name。
    就上面的例子，從root到pointLightShape1的路徑會表示為 “|pCube1|pointLight1|pointLightShape1”
  • MDagPath.isInstanced()/MDagPath.instanceNumber() > 當我們想要同一個shape可以在不同地點呈現，為了減少記憶體消耗，同一個shape可以不用透過複製的方式，我們的作法是透過同一個shape node的上層可以有多個transform nodes，如此一來，從root到該個shape node卻有多個paths。

Traversing the Scene Graph

  • MItDag/MItDependencyGraph > Traversing the Scene Graph
  • Classes prefixed with MIt > iterators
  • MItDag > iterate over the DAG nodes in the scene.
  • OpenMaya.MItDag.MItDag會建造一個DAG iterator然後從root開始用depth-first方法遍歷scene。
  • OpenMaya.MFn.kInvalid確保MItDag不會跳過任何的node type，換成其他參數就會有Filter的功能。
  • MItDag.isDone()判斷是否已經遍歷完所有node
  • MItDag.currentItem() > return the iterator’s current DAG object
  • MItDag.depth() > return currentItem和root之間的depth

Inspecting Selected Scene Elements

  • MGlobal > 是一個static class，提供有關Maya application, logging, object selection, command execution, 3D views (including the scene’s up-axis), and model manipulation.
  • MGlobal.getActiveSelectionList() > 當objects從Maya’s user interface (or through scripts)被選擇時，他們會被加入到global active selection list，而我們可以透過此function來存取。
