程式說明

要創建DAG nodes還有綁定undoing操作，Maya API提供了MDagModifier class。
MDagModifier繼承自MDGModifier，MDGModifier是用來創建及操作Dependency Graph nodes。
前面的章節有說過，DAG是Dependency Graph的一部份，所以MDGModifier有的東西MDagModifier都有（繼承）。

Plug-in

  1. MDagModifier.createNode()
    當呼叫的時候會把他放到MDagModifier的queue，並且此function會回傳一個MObject，這個MObject用來handle新建立的Maya DAG node，但此node還未被included into DAG（要等MDagModifier.doIt()的時候）。
    MDagModifier.createNode()可以有一個string參數，表示node type’s name，允許的參數可以參考 Maya User Guide > Technical Documentation > Nodes
    另外還有MDagModifier.renameNode()，呼叫他一樣會被放到MDagModifier的queue裡，用來renaming of a node。

  2. MDagModifier.doIt()
    會執行之前在queue中的operations，比方說operation是MDagModifier.createNode()，就會完成建立一個node到DAG的操作，在此之前都事先放到queue裡，並沒有真的執行。
    所以通常會把MDagModifier.doIt()放在redoIt()中，這樣一來就可以更有效率的建立node。
    NOTE： 在同一個command中可以呼叫很多次MDagModifier.doIt()

  3. MDagModifier.undoIt()
    所有由MDagModifier.doIt()完成的operations將會被undone。
    NOTE： 建議MDagModifier.undoIt()只呼叫一次，儘管MDagModifier.doIt()可以呼叫很多次。
